# Shonen pack — prompt reference

**API setting for likeness:** always pass `input_fidelity="high"` on the images/edit call — confirmed as OpenAI's actual mechanism for preserving facial identity, unlike submitting multiple reference photos (tested/researched: no evidence multi-angle input improves likeness, and it costs more per generation). Stick to one clear, well-lit, front-facing selfie in onboarding.

**Consistency fix (confirmed no seed/reproducibility control exists on this API):** generate n candidates per sticker per order, auto-validate (or manually pick during testing), deliver the best one. Same prompt does NOT reliably give the same quality every run — this is expected API behavior, not a prompt problem.

**Locked cost approach: low-quality drafts, one final render.** Generate candidates at `quality="low"` ($0.009/image) to pick the winner, then re-render only that winner at `quality="medium"` ($0.034/image). See [project_automation_qa_pipeline_spec] memory for the full n=6 + automated defect-check design (not yet built).

**Two composition modes (confirmed by testing):**
- **Reaction/comedy stickers** (crying, sweatdrop, ghost-leaving-body, nosebleed, pleading, glistening eyes) → extreme close-up, head only cropped at the neck, no arms/hands/shoulders. Full-body attempt on sticker #1 came out distorted/low quality — cropping tight fixed the composition problem.
- **Action/power stickers** (sword, power-up, sports) → full-body dynamic pose is fine, already tested and worked well.

## Why the process is now faster per sticker

Stickers #1's back-and-forth testing (nose/lip shine, teeth color, skin tone lightening bias, stray tear droplets, head angle, crop, hair rendering) surfaced a set of **recurring rendering defects that have nothing to do with which pose/expression is being drawn** — they're API/model biases that show up on every sticker regardless of content. Those fixes are now split out into one reusable **QUALITY CORE** block below, locked once and reused verbatim on every sticker/pose/pack going forward.

This means testing sticker #2 onward should only require iterating on the **pose-specific sentence** (expression + hand/body position) — the identity, skin tone, nose/lip/teeth, hair, crop, and angle rules don't need to be re-litigated or re-tested from scratch each time. If a new defect type shows up on a later sticker that's clearly pose-independent (e.g. a rendering bug that would show up on any sticker), add it to QUALITY CORE once rather than patching each sticker's prompt individually.

Practical speedup for the remaining 16:
1. Write only the `[POSE]` sentence per sticker (expression + explicit hand/arm position — required per the Rule below).
2. Drop it into the template unchanged.
3. Test across 2-3 varied selfies (different skin tones, at least one masculine + one feminine) in the same pass, not sequential single-selfie loops — most of the remaining defects we found (skin tone, teeth color) only surfaced on certain skin tones, so single-selfie testing missed them for several rounds.
4. Only escalate a prompt edit if the defect appears to be pose-specific (e.g., a hand gesture rendering wrong) — pose-independent defects go into QUALITY CORE, not into that sticker's prompt alone.

## QUALITY CORE (locked, reuse verbatim on every sticker/pack)

Insert this block into every prompt, gender-swapped only where noted. Covers: identity/feature preservation, skin tone, nose/lip/teeth rendering, hair rendering, crop, and the final-check catch-all.

> Transform the person in this photo into anime style, keeping their facial structure and overall identity clearly recognizable as the same person. STRICT RULE on facial features: preserve their actual nose, lip, and eye proportions from the photo, stylized at the same degree of anime simplification used for any face — do not narrow, thin, or "correct" features toward a generic anime default, and do not exaggerate them either. STRICT RULE on skin tone: identify the person's true base skin tone directly from the darkest, most shadow-neutral area of their actual face in the photo (jawline, under the cheekbone) — not an average of the whole face, and never a bright highlight/glare area, those are NOT the true skin color. Reproduce that exact tone faithfully, whether it is light, medium, tan, brown, or deep dark brown — do not shift it lighter, warmer, or more "generic" than what the photo shows. The model's known failure mode is defaulting any skin tone toward a lighter medium-brown/tan midtone regardless of the source photo; actively resist that pull and match the real photo's tone instead. When cel-shading, keep all shadow and highlight variations within that same base color family via hue shifts (warm reds/oranges or cool blues), never lightening the base tone itself to make it "cleaner" or more visible. The nose must be rendered completely matte and flat-colored, with no shine, highlight, glint, or specular dot on the tip or bridge — flat cel-shading only, no nose highlight of any kind. This is a small sticker so the nose must read as clean, flat color with zero extra marks. The mouth and lips must also be rendered completely matte and flat-colored — no shine, gloss, highlight dot, or specular mark on the lips, teeth, or inside the mouth. Flat cel-shading only on the entire face: nose, lips, and cheeks all stay shine-free. The visible top row of teeth must be rendered bright white or off-white, with only light gray or pale blue used for subtle shading between individual teeth for depth — never black, dark gray, or a discolored/dirty tone. Teeth must read as clearly white in the final image. STRICT RULE on hair: identify the general category of the person's natural hair texture from the photo (straight, wavy, curly, or coily) and use it as the foundation — do not literally replicate the exact cut or length from the photo. Design a fully realized, attractive [GENDER] hairstyle true to that texture category (for coily hair: voluminous defined coils, twists, or braids — not one generic afro shape; for curly: full defined curls; and so on), built as a shaped, layered mass with real volume, styled as if deliberately done. Render the hair in completely flat, matte cel-shading — no shine, no gloss, no specular highlight strokes, no white rim-light or glow around the silhouette. Use only flat color and subtle tonal shifts within the same hair color family for depth between clumps/curls, exactly like the skin and face shading. STRICT RULE on outline: the only outline in the image is the single bold black sticker border traced tightly around the head/hair silhouette — there must be no additional white or light-colored rim-light, halo, glow, or aura effect surrounding the head, and no soft gradient/blur ring around the outside of the hair. The area immediately outside the black outline is plain transparent background, nothing else. [GENDER-SPECIFIC FACIAL HAIR/FEMINIZATION CLAUSE — see below] [POSE — must explicitly describe both face/expression AND exact hand/body position]. If the person is wearing accessories such as hats, glasses, or jewelry, omit them. [CROP — head-only extreme close-up for reaction/comedy stickers; full-body dynamic for action/power stickers, per composition modes above]. Simple bold clean linework, soft flat cel-shaded coloring, minimal detail. No text, no logos, no watermarks. Output as a clean sticker: transparent background, bold outline, square, high resolution. FINAL CHECK before rendering: the face, nose, mouth, teeth, cheeks, chin, and hair must be completely clean, flat, and matte, with no extra dots, beads, spots, moles, freckles, shine, glow, halo, rim-light, discoloration, or marks anywhere that aren't part of the reference photo or explicitly described in the pose above. Confirm there is exactly one bold black outline around the head silhouette and nothing else surrounding it.

**[GENDER] substitution:**
- Masculine: `masculine` — and the facial-hair clause is: *"If the person has facial hair (beard, goatee, mustache, stubble) in the reference photo, preserve it in the same style and coverage — do not add facial hair if none is present, and do not remove it if present. Bold and dynamic, spiky angular hair where texture allows, sharp confident linework, bright saturated colors, rendered in a masculine anime style, dramatic ink linework, cel-shaded coloring."*
- Feminine: `feminine` — and the clause is: *"STRICT RULE on facial hair: remove any beard, goatee, or mustache completely — the result must be a fully feminine, clean-shaven face with no trace of facial hair. Render them with a deliberately, visibly feminine anime look — this must read unmistakably as a feminine character, not an ambiguous or masculine-leaning one: the jawline must be noticeably narrower and rounder than the reference photo's jaw, the chin softer and less angular, eyebrows thinner and higher-arched than the reference, long dark fluttery eyelashes clearly visible even with closed eyes, fuller lips, and smoother/rounder cheeks. The hairstyle must read as noticeably longer and fuller than the reference photo's actual length — even if the reference hair is short or cropped, style it into a longer, more voluminous feminine cut within the same texture category. This is a comedic gender-swapped anime version of the person — the transformation should be obvious at a glance, not subtle."*

## Pack style modifiers (swap per pack)

- **Shonen:** bold and dynamic, spiky angular hair, sharp confident linework, bright saturated colors
- **Shojo:** soft and dreamy, rounded gentle features, sparkling large eyes, pastel color accents
- **Seinen:** realistic proportions, gritty mature linework, muted grounded color palette
- **Josei:** elegant refined linework, sophisticated realistic style, soft mature tones

## Gender style options (customer picks, not inferred)

- `masculine`
- `feminine`

Customer selects this at purchase/upload — the AI never assumes from the selfie's apparent presentation.
